# cinematic-portfolio-architect

## Trigger
Use this skill when the user asks for:
- overall portfolio direction
- homepage or intro experience planning
- section and route architecture
- scene-to-content mapping
- cockpit, galaxy, or planet-based experience design
- premium portfolio redesign strategy
- how to phase the build from concept to shippable implementation

Do not use this skill for narrow low-level rendering fixes unless the question affects overall experience structure.

## Optimize For
- immersion without confusion
- strong hierarchy between world, navigation, and content
- original but shippable concepts
- clear section identities
- premium pacing and composition
- route and component architecture that can survive iteration
- conversion and trust, not just spectacle

## Operating Rules
- Define the user journey first: arrival, orientation, exploration, deep read, contact.
- Map each scene element to a communication job.
- Keep the world model understandable in one sentence.
- Treat the 3D scene and HTML overlay as separate but coordinated systems.
- Prefer a small number of iconic interaction moments over constant novelty.
- If the concept is strong but too expensive, simplify the rendering rather than flattening the narrative.

## Anti-Patterns
- vague “space theme” with no information architecture
- sections that exist only as visual gimmicks
- forcing every interaction through the 3D layer
- cinematic sequences that block access to content
- adding lore instead of clarity
- homepages that feel like demos instead of portfolios

## Required Output Structure For Medium And Large Tasks
### Diagnosis
- What is unclear, generic, structurally weak, or disconnected.

### Proposed Direction
- The experience concept, section logic, and scene-to-content strategy.

### Implementation Plan
- The practical sequence of changes.

### Files To Change
- The files or folders to create or modify.

### Code
- The implementation or scaffold.

### Validation
- What should be checked after implementation.

### Open Risks
- What may still need refinement or staged follow-up.

## Quality Threshold
- The result should feel like a designed experience system, not a themed skin on a normal portfolio.
- If the concept cannot be explained clearly, it is not ready to build.
